Ruby Classes

Classes vs Objects

As you may already know, Ruby is object-oriented and so everything in Ruby is an object. To understand what that really means, let's go over the difference between an object and a class. Classes define the behaviors of the objects in Ruby. A class is like a blueprint, once it's created you can use it to build as many "houses" as you'd like. Objects are instances of a class and have their own data, each object "remembers" it's class and the parameters that go with it. For example, a string is a class and an object within that class could be the specific string "hello!".
