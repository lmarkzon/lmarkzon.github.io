Ruby Classes

Classes vs Objects

As you may already know, Ruby is object-oriented and so everything in Ruby is an object. To understand what that really means, let's go over the difference between an object and a class. Classes define the behaviors of the objects in Ruby. A class is like a blueprint, once it's created you can use it to build as many "houses" as you'd like. An Object is an instance of a class, it has its own data and "knows" its class, including the behaviors that go with it. When objects are defined inside of a method, they are scoped to the method, and disappear when the method ends. For example, a string is a class and an object within that class could be the specific string "hello!".

Instance and Class Variables

Variables contain the data that's stored in objects. When variables are defined inside of a method, they are also scoped to the method. They come into existence brand new each time the method runs, and then disappears when the method ends. Instance Variables, prefixed @instancevariable with one symbol, and are scoped to the class. They can be accessed from any method within that class, and keep their value between method calls. Class Variables are tied to the class so there is only one total value, opposed to how Instance Variables have one for each and every instance/occurrence. Class variables are prefixed @@classvariables with two symbols instead of one, and are not as common as instance variables. To create an Instance you can call a method on the class new, such as "string = String.new". Objects of most classes can be created by also calling new(). Instance and Class Methods

Instance Methods are called through a particular instance, such as “Hello”.size( ), and can access pretty much anything in its class. This includes reading and writing instance and/or class variables and calling other instance methods or class methods. They also have read-access to a special value called “self", which keeps a reference of the particular object the method was called on. Class Methods are called through the particular class, so you don't need an instance, and they're less common than instance methods. They are prefixed with the class name or 'self'.. Since you don't need an instance to call these methods, they can't access instance variables/methods, can't refer to 'self', they can only operate on class variables and whatever arguments are supplied to them when they're called.

Special Instance Method: Initialize

There is a Special Instance Method called a Constructor and different coding languages all have different mechanisms for defining them. In ruby it's a special instance method called initialize(), when an instance of the class is created, initialize automatically runs. Parameters can be added to initialize, and you'll also need to pass those parameters to the constructor 'new' like this: my_array = Array.new(5). Five will be passed to initialize and this will create an array with 5 nil elements. The purpose of this is to do any "set up" needed, usually sets the starting values for instance variables.
